## 简介

> **GNU 编译器套装**（英语：GNU Compiler Collection，缩写为 GCC）是 GNU 计划[^1]制作的一种优化编译器，支持各种编程语言、操作系统、计算机系统结构。该编译器是以 GPL 及 LGPL 许可证所发行的自由软件，也是 GNU 计划的关键部分，还是 GNU 工具链的主要组成部分之一。GCC（特别是其中的 C 语言编译器）也常被认为是跨平台编译器的事实标准。 **_----Wikipedia_**

GCC 包含 gcc、g++等编译器，同时这个工具集不及包含编译器，还包含其他工具集，如 ar、nm 等（可在对应平台的 bin 目录下查看）。

GCC 工具集不仅能编译 C/C++ 语言，其他例如 Objective-C、Pascal、Fortran、Java、Ada 等语言均能进行编译。GCC 在可以根据不同的硬件平台进行编译，即能进行交叉编译，在 A 平台上编译 B 平台的程序，支持常见的 X86、ARM、PowerPC、mips 等，以及 Linux、Windows 等软件平台。

在 Windows 下，**MinGW**（Minimalist GNU for Windows），又称 mingw32，就是将 GCC 编译器和 GNU Binutils 移植到 Win32 平台下的产物，包括一系列头文件（Win32API）、库和可执行文件。另有可用于产生 32 位及 64 位 Windows 可执行文件的 MinGW-w64 项目，是从原本 MinGW 产生的分支。

[^1]: GNU 计划（英语：GNU Project），又译为革奴计划，是一个自由软件集体协作计划，1983 年 9 月 27 日由理查德·斯托曼在麻省理工学院公开发起。它的目标是创建一套完全自由的操作系统，称为 GNU。 **_----Wikipedia_**

## gcc 与 g++

关于 gcc 和 g++很多人的理解都是比较片面的，或者对两者有什么误解。下面从三个方面来解释一下二者的不同：

1. 代码编译方面：
   - 后缀为 `.c` 的文件，gcc 把它当作 C 程序，而 g++当作 C++程序。
   - 后缀为 `.cpp` 的文件，二者都认为是 C++程序。
   - g++会调用 gcc，对于 c++代码，二者是等价的，也就是说 gcc 和 g++都可以编译 C/C++程序。
2. 链接阶段：
   - gcc 和 g++都可以自动链接到标准 C 库
   - g++可以自动链接到标准 C++库，而 gcc 要链接到标准 C++库则需要加参数 `-lstdc++` 。
3. 关于 `_cplusplus` 宏定义
   - g++会自动定义该宏，但是这个不影响它去编译 C 程序。
   - gcc 需要根据文件后缀判断是否需要定义该宏。

```shell
# 编译 c 程序
$ gcc test.c -o test
$ g++ test.c -o test

# 编译 c++ 程序
$ gcc test.cpp -lstdc++ -o test
$ g++ test.cpp  -o test
```

## 安装

有些纯净版的 Linux 默认没有 gcc 编译器，需要自己安装，步骤如下：

```shell
# 安装软件必须要有管理员权限
# ubuntu
$ sudo apt update   		# 更新本地的软件下载列表, 得到最新的下载地址
$ sudo apt install gcc g++	# 通过下载列表中提供的地址下载安装包, 并安装

# centos
$ sudo yum update   		# 更新本地的软件下载列表, 得到最新的下载地址
$ sudo yum install gcc g++	# 通过下载列表中提供的地址下载安装包, 并安装
```

安装后可用如下命令查看版本：

```
gcc -v  or  gcc --version
g++ -v  or  g++ --version
```

## 工作流程

GCC 编译器的工作流程如下图所示，共分 4 个阶段：**预处理（预编译）**、**编译和优化**、**汇编**、**链接**。

1. 预处理：这个阶段主要做三件事：**展开头文件**、**宏替换**、**去掉注释行**。GCC 调用预处理器完成，最终得到的还是源文件（处理后的）。
2. 编译：GCC 调用编译器对文件进行编译，最终得到汇编文件。
3. 汇编：GCC 调用汇编器对文件进行汇编，最终得到二进制文件。
4. 链接：GCC 调用链接器对程序所需的库进行链接，最终得到可执行的二进制文件。

```mermaid
graph LR
    A[源文件]--预处理-->B[预处理后的源文件]--编译-->C[汇编文件]--汇编-->D[目标文件]--链接-->E[可执行程序]
```

| 文件名后缀 |         说明         | GCC 参数 |
| :--------: | :------------------: | :------: |
|    `.c`    |        源文件        |    无    |
|    `.i`    |   预处理后的源文件   |   `-E`   |
|    `.s`    | 编译后得到的汇编文件 |   `-S`   |
|    `.o`    | 汇编后得到的目标文件 |   `-c`   |

GCC 的编译器可以将这四步合并成一个。如在 Linux 下编译单个文件十分简单，直接使用 gcc 命令即可，GCC 就会自动生成后缀为 `.out` 的可执行文件。用一个命令就可以完成这四步工作，当然也可以一步一步地执行。

以 `test.c` 文件为例：（ `-o` 指定输出文件名）

```shell
$ gcc -E test.c -o test.i   #预处理
$ gcc -S test.i -o test.s   #编译
$ gcc -c test.s -o test.o   #汇编
$ gcc test.o -o test        #链接（不需要任何参数）
```

## 常用参数

参数在 gcc 命令中没有位置要去，只要按需指定出来即可。

|  gcc 编译选项   |                    意义                    |
| :-------------: | :----------------------------------------: |
|      `-E`       |        指定要预编译的源文件，不编译        |
|      `-S`       |          指定要编译的文件，不汇编          |
|      `-c`       |          指定要汇编的文件，不链接          |
|      `-o`       |               指定输出文件名               |
| `-I` (大写的 i) |      指定 include 包含文件的搜索路径       |
|      `-g`       | 编译时，生成调试信息，该程序可被调试器调试 |
|      `-D`       |                 指定一个宏                 |
|      `-l`       |  指定编译时使用的库,直接加库名(不加空格)   |
|      `-L`       |           指定编译时库的搜索路径           |
|     `-Wall`     |   允许发出 Gcc 提供的所有有用的报警信息    |

`-I` 示例：

```shell
# 示例目录结构
    .
    |---main.cpp
    |---include/------->src.h
    |---src.cpp

# 编译命令
gcc -I ./include -o main *.cpp
```

`-D` 示例：
在程序中我们经常通过定义一个宏来调整编译过程（利用 `#ifdef` 、 `#ifndef` 、 `#endif` 等等），如下 `test.cpp` ，我们通过 `TEST` 这个宏来控制是否编译 `cout` 输出内容。

```cpp
#include <iostream>

int main()
{

#ifdef TEST
    std::cout << "hello world!" << std::endl;
#endif

    return 0;
}
```

但是我们如果不想在程序中定义宏，但又想让它存在的话，我们就可以通过 gcc 的参数 `-D` 传给编译器一个宏以实现相应的功能。

```shell
# 在编译命令中定义TEST宏
$ gcc test.cpp -D TEST -o test

# 执行程序test，观察输出
$ ./test
hello world!
```

`-D` 参数的应用场景:

- 在发布程序的时候，一般都会要求将程序中所有的 log 输出去掉，如果不去掉会影响程序的执行效率，很显然删除这些打印 log 的源代码是一件很麻烦的事情，解决方案是这样的：
  - 将所有的打印 log 的代码都写到一个宏判定中，可以模仿上边的例子
  - 在编译程序的时候指定 `-D` 就会有 log 输出
  - 在编译程序的时候不指定 `-D`, log 就不会输出

`-l` 示例
- `-l` 参数就是用来指定程序要链接的库，`-l` 参数紧接着就是库名（不加空格），那么库名跟真正的库文件名有什么关系呢？就拿数学库来说，他的库名是 `m`，他的库文件名是 `libm.so`，很容易看出，把库文件名的头 `lib` 和尾 `.so` 去掉就是库名了,链接参数应写成 `-lm`。

## `-l` 与 `-L`
- `-l` 参数上面已经解释过了，若没有 `-L` 参数，`-l` 只能链接本地库目录或者系统库目录中的库（Linux平台下路径为 `/lib`、`/usr/lib` 和 `/usr/local/lib`），即若链接参数仅为 `-lm`，则编译器只会在`/lib`、`/usr/lib` 和 `/usr/local/lib` 目录中寻找 `libm.so` 库文件。
- 我们要想链接第三方库，则需要为编译器指定库文件的搜索路径，即使用 `-L` 参数，比如我们把 `libtest.so` 放到 `/a/b/c` 目录下，那么链接参数应该写成如下形式：
```
-L /a/b/c -ltest
```

## 多文件编译

GCC 可以自动编译链接多个文件，不管是目标文件还是源文件，都可以使用同一命令编译到一个可执行文件中。这极大方便了我们模块化工作。

因为头文件时包含在源文件中的，因此在使用 gcc 编译程序时不需要指定头文件的，当头文件无法被找到时使用参数 `-I` 指定其具体路径即可。示例同上。
